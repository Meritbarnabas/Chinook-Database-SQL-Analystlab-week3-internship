# Chinook Database SQL Analysis

## Project Overview

This project was completed as part of Week 3 of the AnalystLab Africa Data Analytics Internship Program. The objective was to apply SQL concepts to extract, transform, and analyze data from the Chinook Music Store Database.

## Dataset

The Chinook database is a sample music store database containing information about:

* Customers
* Employees
* Invoices
* Invoice Lines
* Tracks
* Albums
* Artists
* Genres

## Data Preparation

The following data cleaning steps were performed:

* Replaced NULL values in Composer, Company, State, Fax, PostalCode, BillingState, and BillingPostalCode columns.
* Converted BirthDate, HireDate, and InvoiceDate fields to DATE format where appropriate.
* Validated table relationships and primary/foreign keys.

## SQL Concepts Applied

* SELECT
* WHERE
* ORDER BY
* GROUP BY
* HAVING
* Aggregate Functions (SUM, AVG, COUNT)
* INNER JOIN
* Subqueries
* Window Functions (RANK, ROW_NUMBER, PARTITION BY)
* Index Creation and Query Optimization

## Key Business Questions Answered

1. Which countries generate the most revenue?
2. Who are the highest-spending customers?
3. Which tracks and genres perform best?
4. What are the revenue trends over time?
5. Which customers spend above average?

## Key Findings

* USA generated the highest revenue ($523.06).
* Helena Holý was the highest-spending customer.
* Rock was the best-selling genre with 835 units sold.
* Revenue remained stable throughout most of 2021.
* 59 customers spent above the average transaction value.
